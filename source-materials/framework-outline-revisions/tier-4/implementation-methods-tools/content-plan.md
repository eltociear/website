## **CONTENT STANDARDS & GUIDELINES**

### **Writing Style**
- **Tone**: Practical, accessible, culturally sensitive
- **Audience**: Implementation practitioners, community organizers
- **Format**: Structured with clear sections, examples, checklists
- **Length**: Varies by content type (see individual specifications)

### **Required Sections for Each File**
1. **Front matter**: Title, section, status, revision info
2. **Overview**: Purpose and scope
3. **Content sections**: Specific to file type
4. **Integration points**: Links to other frameworks/tools
5. **Implementation guidance**: Practical next steps
6. **Cultural considerations**: Adaptation protocols
7. **Resources**: Additional references and tools

### **Cultural Integration Requirements**
- **Anti-colonial evaluation**: Every tool must pass evaluation
- **Indigenous protocols**: FPIC 2.0 integration where applicable
- **Language justice**: Multi-lingual considerations
- **Traditional knowledge**: Respectful integration protocols
